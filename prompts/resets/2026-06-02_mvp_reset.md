## Vision & Core Philosophy

Mycelium creates **AI-native, self-managing data sources** where **100% of the infrastructure is managed by AI agents** (not humans). External agents use the MCP server (JSON).

**All data is agent-owned** (including the "core"):
- The **Supervisor** is a pure **coordinator/router** — never a data owner or direct accessor.
- Specialist agents own both responsibility *and* their own storage strategy.
- Even identity resolution may route through specialists.
- No pre-defined derivatives or storage structures — they emerge from agents.
- No god-agents; supervisor stays narrow and explicit.

**Phase 1 Rule:** Keep shared storage dead simple. Supervisor direct storage access is a temporary concession only.

See `docs/architecture.md` (the living source of truth) for full details.

## Phase 1 MVP — Strictly Minimal Core

**Core Data Model:**
- `Person`: **only** `id`, `name`, `employer`
- `CORE_PERSON_FIELDS = {"id", "name", "employer"}`
- `MINIMUM_VIABLE_FIELDS = ["name", "employer"]`
- Everything else is non-core and routed to specialists.

**Key Architecture (see `docs/architecture.md` for depth):**
- **Supervisor** (`src/agents/supervisor.py`): thin coordinator. Delegates to `routing.py`, `responses.py`; core access via **CoreIdentity** (`src/agents/core_identity.py` — Phase 1 specialist facade for the `people` table). Future: full autonomous specialist.
- **PersonResponse** (external contract): `results` (core dicts), `message` (narrative for agents/humans), `debug` (internal), `trace_id` (LangSmith when enabled), `thread_id` (sessions + LangGraph checkpoints).
- **Core Ingestion Handshake**: Single-step via `provided_data`. Enrich → validate → persist *only on success*.
- **Observability**: `trace_id` + `thread_id` set in `run_query` (post-graph).
- **MCP**: `src/mycelium_mcp/` (renamed to avoid `mcp` SDK collision). Entry: `mycelium-mcp = "mycelium_mcp.server:run_server"`.
- **Tech**: LangGraph (explicit + SQLite checkpointer), SQLite (data + checkpoints), FastMCP, Pydantic v2, strict typing, LangSmith from day one.

Enrich/validator are still graph nodes for ingest.

## Collaboration & Cursor Workflow

- **Paul**: Vision/priorities.
- **Grok**: Planning, architecture, reviews, prompts.
- **Cursor**: Senior developer executing via prompts.

**Cursor Handoff** (see `prompts/cursor/WORKFLOW.md` + `.cursor/rules/04-cursor-workflow.mdc`):

- `next/`: Ready tasks (`YYYY-MM-DD-HHMM-slug.md`)
- `in-progress/`: Claimed (the lock)
- `done/`: Artifacts (`prompt.md`, `output.md`, `review.md`)

**When told "Work on the next task":**
1. Scan `next/`, sort alphabetically (oldest first).
2. **Immediately move** first file to `in-progress/` (claim).
3. Execute per prompt (strict scope).
4. Deliver to `done/<name>/`.
5. **Remove only your claimed file** from `in-progress/`. Never touch others (parallel safety).

**Scope Discipline:** Out-of-scope? Stop, document, create follow-up prompt.

**Cursor Config (reproducibility):**
- `.cursor/rules/`: langgraph, python, workflow, etc.
- `.cursor/hooks/` + `hooks.json`: auto-allow file deletes; gate `rm -r`/dir deletions.
